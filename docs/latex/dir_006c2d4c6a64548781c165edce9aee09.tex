\hypertarget{dir_006c2d4c6a64548781c165edce9aee09_autotoc_md24}{}\doxysubsubsection{\texorpdfstring{How it works}{How it works}}\label{dir_006c2d4c6a64548781c165edce9aee09_autotoc_md24}

\begin{DoxyEnumerate}
\item First the application creates a {\bfseries{window}} calling {\bfseries{mfb\+\_\+open}} or {\bfseries{mfb\+\_\+open\+\_\+ex}}.
\item Next, it\textquotesingle{}s the application\textquotesingle{}s responsibility to allocate a buffer to work with.
\item Next, call {\bfseries{mfb\+\_\+update}} or {\bfseries{mfb\+\_\+update\+\_\+ex}} to copy the buffer to the window and display it. If this function returns a value lower than 0, the window has been destroyed internally and cannot be used anymore.
\item Last the code waits to synchronize with the monitor calling {\bfseries{mfb\+\_\+wait\+\_\+sync}}.
\end{DoxyEnumerate}

{\bfseries{Note:\+}} By default, if ESC key is pressed, {\bfseries{mfb\+\_\+update}} /\+ {\bfseries{mfb\+\_\+update\+\_\+ex}} will return -\/1 (and the window will have been destroyed internally).

{\ttfamily mfb\+\_\+set\+\_\+viewport()} returns {\ttfamily false} if:\+
\begin{DoxyItemize}
\item {\ttfamily width == 0} or {\ttfamily height == 0}
\item viewport bounds exceed the current window drawable size
\end{DoxyItemize}

{\ttfamily mfb\+\_\+open()} and {\ttfamily mfb\+\_\+open\+\_\+ex()} return {\ttfamily NULL} if:\+
\begin{DoxyItemize}
\item {\ttfamily width == 0} or {\ttfamily height == 0}
\item {\ttfamily width \texorpdfstring{$\ast$}{*} 4} would overflow the internal framebuffer stride
\end{DoxyItemize}

If {\ttfamily title} is {\ttfamily NULL} or empty, Mini\+FB uses {\ttfamily "{}minifb"{}} as the effective window/\+canvas title.

If both {\ttfamily MFB\+\_\+\+WF\+\_\+\+FULLSCREEN} and {\ttfamily MFB\+\_\+\+WF\+\_\+\+FULLSCREEN\+\_\+\+DESKTOP} are provided, {\ttfamily MFB\+\_\+\+WF\+\_\+\+FULLSCREEN} takes precedence.

{\ttfamily mfb\+\_\+update\+\_\+ex()} returns {\ttfamily MFB\+\_\+\+STATE\+\_\+\+INVALID\+\_\+\+BUFFER} if:\+
\begin{DoxyItemize}
\item {\ttfamily buffer == NULL}
\item {\ttfamily width == 0} or {\ttfamily height == 0}
\item {\ttfamily width \texorpdfstring{$\ast$}{*} 4} would overflow internal stride calculations
\end{DoxyItemize}

{\ttfamily mfb\+\_\+update\+\_\+ex()} runtime behavior is backend-\/specific:\+
\begin{DoxyItemize}
\item Wayland waits for compositor frame callback inside {\ttfamily mfb\+\_\+update\+\_\+ex()} (can block).
\item Android may return {\ttfamily MFB\+\_\+\+STATE\+\_\+\+OK} without presenting when {\ttfamily ANative\+Window} is temporarily unavailable during lifecycle transitions.
\end{DoxyItemize}

Open-\/time readiness is backend-\/specific:\+
\begin{DoxyItemize}
\item Wayland waits for the initial configure handshake before returning from {\ttfamily mfb\+\_\+open\+\_\+ex()}.
\item Android may return a window handle before {\ttfamily ANative\+Window} is ready (rendering starts once the native window becomes available).
\end{DoxyItemize}

Mini\+FB existed for many years without an official release version. Starting from this codebase, the project now uses Sem\+Ver and declares {\bfseries{v0.\+9.\+0}} as the official baseline version.

Builds expose public version/\+build metadata for C/\+\+C++ consumers via {\ttfamily \doxylink{minifb__version_8h_source}{minifb\+\_\+version.\+h}} (generated at configure time and installed with the public headers). This includes:\+

The web backend has the following backend-\/specific behavior:\+


\begin{DoxyItemize}
\item In {\ttfamily mfb\+\_\+open\+\_\+ex()}, only fullscreen flags are currently interpreted ({\ttfamily MFB\+\_\+\+WF\+\_\+\+FULLSCREEN}, {\ttfamily MFB\+\_\+\+WF\+\_\+\+FULLSCREEN\+\_\+\+DESKTOP}); other window flags are ignored
\item {\ttfamily mfb\+\_\+get\+\_\+monitor\+\_\+dpi()} /\+ {\ttfamily mfb\+\_\+get\+\_\+monitor\+\_\+scale()} report fixed values ({\ttfamily 1.\+0} scale)
\item {\ttfamily mfb\+\_\+set\+\_\+target\+\_\+fps()} /\+ {\ttfamily mfb\+\_\+get\+\_\+target\+\_\+fps()} store/\+query the target value, but do not currently control browser frame pacing (the browser event loop /\+ RAF timing drives pacing)
\item {\ttfamily mfb\+\_\+show\+\_\+cursor()} tracks requested visibility, but hiding the browser cursor is not implemented yet
\end{DoxyItemize}

Core rendering, events, viewport and timers are supported.

When calling {\ttfamily mfb\+\_\+open()} or {\ttfamily mfb\+\_\+open\+\_\+ex()}, Web uses the effective title as canvas id.
