\begin{DoxyItemize}
\item {\bfseries{Logging API}}:\+ {\ttfamily mfb\+\_\+set\+\_\+logger}, {\ttfamily mfb\+\_\+set\+\_\+log\+\_\+level}, {\ttfamily mfb\+\_\+log}, {\ttfamily mfb\+\_\+log\+\_\+level}, {\ttfamily \doxylink{structmfb__log__info}{mfb\+\_\+log\+\_\+info}}, and {\ttfamily MFB\+\_\+\+LOG\texorpdfstring{$\ast$}{*}} helper macros for runtime log control and source-\/location-\/aware diagnostics. \item {\bfseries{Display inset APIs}}:\+ {\ttfamily mfb\+\_\+get\+\_\+display\+\_\+cutout\+\_\+insets} and {\ttfamily mfb\+\_\+get\+\_\+display\+\_\+safe\+\_\+insets} for mobile-\/safe layouts (Android API 28+, i\+OS, desktop stubs return zeros).
\item {\bfseries{Touch pointer decoding}}:\+ {\ttfamily mfb\+\_\+decode\+\_\+touch}, {\ttfamily mfb\+\_\+decode\+\_\+touch\+\_\+pos}, and {\ttfamily mfb\+\_\+decode\+\_\+touch\+\_\+id} to decode packed pointer id/\+position values from mobile mouse getters.
\item {\bfseries{Monitor scale}}:\+ implemented {\ttfamily mfb\+\_\+get\+\_\+monitor\+\_\+scale} for Web ({\ttfamily device\+Pixel\+Ratio}) and Android.
\item {\bfseries{Cursor control}}:\+ implemented {\ttfamily mfb\+\_\+show\+\_\+cursor} for Web.
\item {\bfseries{X11 scale detection}}:\+ layered fallbacks (XSettings, Xresources, XRandR, physical DPI).
\item {\bfseries{DOS viewport}}:\+ basic viewport support for the MS-\/\+DOS backend.
\item {\bfseries{Android {\ttfamily mfb\+\_\+update\+\_\+events}}}:\+ event-\/only pump without rendering, matching other backends.
\item {\bfseries{Android example}}:\+ new example project using Android Studio Narwhal (native2026).
\item {\bfseries{New headers}}:\+ {\ttfamily \doxylink{_mini_f_b__macros_8h_source}{Mini\+FB\+\_\+macros.\+h}} (deprecation/\+pixel/\+logging macros), {\ttfamily \doxylink{_mini_f_b__types_8h_source}{Mini\+FB\+\_\+types.\+h}} (callback and logging typedefs), {\ttfamily \doxylink{_window_data___web_8h_source}{Window\+Data\+\_\+\+Web.\+h}}.
\item {\bfseries{Internal helpers}}:\+ {\ttfamily calculate\+\_\+buffer\+\_\+layout} (overflow-\/safe buffer validation) and {\ttfamily mfb\+\_\+validate\+\_\+viewport} (unified viewport checks), used by all backends.
