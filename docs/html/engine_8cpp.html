Detailed Description

Entry point for the CXX Rasterizer — a minimal Wavefront OBJ viewer with a software-rendered MiniFB window.

Loads a Wavefront OBJ file supplied on the command line, logs parse time and heap usage, opens an 800×600 resizable MiniFB window, and renders a small set of test lines using render_v0's Bresenham rasterizer before entering the main display loop.

Dependencies
  • C++20 module obj_v0_3 — OBJ file parsing and in-memory representation.
  • C++20 module render_v0 — 2D primitive rasterization into a pixel buffer.
  • MiniFB — cross-platform framebuffer window library.
Usage
./rasterizer <path/to/model.obj>

Application entry point.

Performs the following steps in order:

  1. Validates that a file path argument was supplied; exits with code 1 otherwise.
  2. Parses the OBJ file at argv[1], logging wall-clock parse time and reserved heap memory via obj::memory_out().
  3. Opens a resizable MiniFB window titled "CXX Rasterizer" at WIDTH × HEIGHT pixels; exits with code 1 on failure.
  4. Allocates a flat WIDTH × HEIGHT pixel buffer initialised to zero.
  5. Constructs three test points and four randomly coloured buff_color instances, then draws four lines between those points into the buffer using render_v0::line().
  6. Enters the MiniFB event loop, pushing the buffer to the window every sync interval until the window is closed or an error state is returned.
Parameters
argcNumber of command-line arguments (must be ≥ 2).
argvArgument vector. argv[1] must be a path to a valid .obj file.
Returns
0 on clean exit, 1 on argument or window-creation error.
Note
The four buff_color instances are default-constructed (all channels 0), which triggers random colour generation in buff_color's constructor. Each line is therefore drawn in a distinct, unpredictable colour.
Warning
The pixel buffer is std::size_t typed to match the MiniFB API on this platform. Ensure mfb_update_ex expects the same element width if porting to a different MiniFB configuration.
